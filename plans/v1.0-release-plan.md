### 2. Fix DNS Resolution Bug (EAI_AGAIN)

**Bug Description:**
The application fails when DNS resolution fails for the Traefik API hostname (`EAI_AGAIN` error). This is a transient network error that should be handled gracefully with retry logic.

**Root Cause:**
- No retry mechanism for transient network failures (DNS, connection refused, timeouts)
- The sync fails repeatedly without exponential backoff
- Application crashes when DNS temporarily cannot resolve the hostname

**Solution:**
- [x] Add retry logic with exponential backoff to network requests
- [x] Handle specific error codes: EAI_AGAIN, ECONNREFUSED, ETIMEDOUT, ENOTFOUND
- [x] Add configurable retry options (max retries, initial delay, max delay)
- [x] Apply retry logic to both TraefikService and PiHoleService

**Implementation Details:**
- [x] Create a utility function for retry with exponential backoff
- [x] Configure default values: 3 retries, 1s initial delay, 30s max delay
- [x] Log retry attempts for debugging
- [x] Add unit tests for retry logic

## Tasks You Will Handle Manually

### Git History Rewrite

To remove sensitive IPs (192.168.1.*) from git history, run these commands locally:

```bash
# Install git-filter-repo first
pip install git-filter-repo

# Clone your repo if you haven't already
git clone https://github.com/YOUR_USERNAME/traefik-pihole-dns-sync.git
cd traefik-pihole-dns-sync

# Replace IP addresses with placeholder
git filter-repo --replace-text --expression '192.168.1.' --replace 'xxx.xxx.xxx.' --force

# Push rewritten history
git push --force --all
git push --force --tags
```

**Warning**: This is a destructive operation. All forks/clones will be broken and need to be re-cloned.
