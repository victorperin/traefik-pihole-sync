## Overview

This release introduces three major features: support for multiple Traefik instances, optional deletion protection to prevent accidental DNS record removal, and a minimal distroless Docker image for security-conscious deployments. This release also includes various bug fixes and improvements to stability.

### Release Highlights

- **Multi-Traefik Connectivity**: Connect to multiple Traefik API endpoints simultaneously and aggregate routers from all instances
- **Deletion Protection**: Optional flag to prevent accidental DNS record deletion in production environments
- **Distroless Image**: Minimal, secure Docker image variant using distroless base (gcr.io/distroless/static:nonroot)
- **Feature Toggles**: All new features are toggleable via environment variables and CLI flags

---

## Objectives

### Objective 1: Multi-Traefik Instance Support

**Description:** Enable the application to connect to multiple Traefik API endpoints simultaneously, aggregate routers from all instances, and display them with clear instance identification in the UI.

**Business Value:** Organizations with distributed Traefik deployments (e.g., edge and core, or multiple data centers) can now manage DNS records from a single dashboard.

**Success Metrics:**

- At least 3 simultaneous Traefik instances supported
- Connection failure to one instance does not affect others
- UI clearly distinguishes routers by source instance

### Objective 2: Deletion Protection

**Description:** Provide an optional mechanism to completely disable DNS record deletion, preventing accidental removal in production environments. This is critical for environments where DNS stability is paramount.

**Business Value:** Prevents production outages caused by accidental DNS record deletion, reducing operational risk.

**Success Metrics:**

- Toggle completely disables delete functionality
- Blocked attempts are logged for audit purposes
- UI provides clear feedback when deletion is disabled

### Objective 3: Distroless Docker Image

**Description:** Create a minimal Docker image variant using distroless base images that contains only the application binary with no shell, package manager, or unnecessary system utilities.

**Business Value:** Reduces attack surface, minimizes image size, and meets security requirements for air-gapped or high-security environments.

**Success Metrics:**

- Image size under 30MB
- No shell or package manager present
- Same functionality as standard image

## Feature Toggles

| Feature             | Environment Variable | CLI Flag             | Default         | Description                                      |
| ------------------- | -------------------- | -------------------- | --------------- | ------------------------------------------------ |
| Multi-Traefik       | `TRAEFIK_INSTANCES`  | N/A                  | Single instance | JSON array of Traefik endpoint configurations    |
| Deletion Protection | `DISABLE_DELETION`   | `--disable-deletion` | `false`         | When `true`, prevents all DNS record deletion    |
| Debug Mode          | `DEBUG`              | `--debug`            | `false`         | Enables verbose logging for troubleshooting      |
| Sync Interval       | `SYNC_INTERVAL`      | `--sync-interval`    | `60000`         | Interval in milliseconds between sync operations |

### Feature Toggle Migration Strategy

For users upgrading from v1.0:

1. **Backward Compatibility**: Single `TRAEFIK_API_URL` continues to work and is automatically converted to the new multi-instance format internally
2. **Graceful Degradation**: If `DISABLE_DELETION` is not set, behavior remains unchanged (deletion allowed)
3. **Migration Guide**: Include migration instructions in release notes

## Rollback Plan

### Criteria for Rollback

- Critical bug affecting more than 10% of users
- Security vulnerability in new features
- Data loss or corruption
- Complete system failure

### Rollback Steps

1. **Application Rollback**:

   ```bash
   # Revert to previous version
   docker pull ghcr.io/victorperin/traefik-pihole-sync:v1.0.0
   kubectl set image deployment/traefik-pihole-dns-sync app=v1.0.0
   ```

2. **Configuration Rollback**:
   - Restore previous `TRAEFIK_API_URL` format
   - Set `DISABLE_DELETION=false` if previously unset

3. **Verification**:
   - Check application logs for errors
   - Verify DNS sync is functioning
   - Confirm deletion works if previously disabled

## Configuration Examples

### Multi-Traefik Configuration (.env)

```env
# Single instance (backward compatible)
TRAEFIK_API_URL=http://traefik:8080
TRAEFIK_API_KEY=your-api-key

# OR multiple instances (new format)
TRAEFIK_INSTANCES=[{"name":"edge","url":"https://edge.local:8080","apiKey":"edge-key"},{"name":"core","url":"https://core.local:8080","apiKey":"core-key"}]
```

### Disable Deletion

```env
# Via environment variable
DISABLE_DELETION=true
```

```bash
# Via CLI flag
./traefik-pihole-dns-sync serve --disable-deletion
```

### Docker Compose with All Features

```yaml
services:
  traefik-pihole-dns-sync:
    image: ghcr.io/victorperin/traefik-pihole-sync:v1.1.0
    container_name: traefik-pihole-dns-sync
    environment:
      - TRAEFIK_INSTANCES=[{"name":"edge","url":"https://edge.local:8080","apiKey":"edge-key"},{"name":"core","url":"https://core.local:8080","apiKey":"core-key"}]
      - PIHOLE_URL=http://pihole:80
      - PIHOLE_PASSWORD=your_password
      - REVERSE_PROXY_IPS=xxx.xxx.xxx.xxx
      - DISABLE_DELETION=true
      - SYNC_INTERVAL=30000
      - LOG_LEVEL=info
    restart: unless-stopped
```

### Kubernetes Deployment (Distroless)

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: traefik-pihole-dns-sync
spec:
  replicas: 1
  selector:
    matchLabels:
      app: traefik-pihole-dns-sync
  template:
    metadata:
      labels:
        app: traefik-pihole-dns-sync
    spec:
      containers:
        - name: app
          image: ghcr.io/victorperin/traefik-pihole-sync:v1.1.0-distroless
          env:
            - name: TRAEFIK_INSTANCES
              value: '[{"name":"edge","url":"https://edge.local:8080","apiKey":"edge-key"}]'
            - name: PIHOLE_URL
              value: "http://pihole:80"
            - name: PIHOLE_PASSWORD
              valueFrom:
                secretKeyRef:
                  name: pihole-credentials
                  key: password
            - name: REVERSE_PROXY_IPS
              value: "xxx.xxx.xxx.xxx"
            - name: DISABLE_DELETION
              value: "true"
          securityContext:
            readOnlyRootFilesystem: true
            runAsNonRoot: true
            runAsUser: 65532
            runAsGroup: 65532
            allowPrivilegeEscalation: false
```
